SRA 是一个运行在 AI Agent 旁的守护进程，在 Agent 收到用户消息时自动分析语义、推荐最匹配的 Hermes Skill，并通过 pre_tool_call 钩子系统在校验工具调用前检查技能遵循度。

🛠️ 开发工作流

强制流程（AGENTS.md）

# 每步开发前必须执行
1. Pre-Flight:   python3 ~/.hermes/scripts/pre_flight.py "<Task>"
2. Skill Discovery: python3 ~/.hermes/skills/.../skill_finder.py "<Keywords>"
3. 如果匹配≥30:  skill_view(name) 加载 skill
4. 开发实现: 实现 → 测试 → 验证
5. 文档对齐 (MANDATORY):
   - skill_view(name="doc-alignment")
   - 执行 5 步文档对齐协议
   - 更新 PROJECT-PANORAMA.html
   - 更新 docs/API-REFERENCE.md (端点变更时)
   - 更新 ROADMAP.md (Story 完成时)
   - 更新 docs/ARCHITECTURE.md (架构变更时)
6. 提交: 代码变更 + 文档变更 = 同一次提交

为什么需要文档对齐？

🧠 上下文污染问题：

文档陈旧是 AI 项目最大的隐患之一。当文档描述与实际代码不一致时：
- AI Agent 加载旧文档 → 基于错误上下文决策 → 产出质量下降
- 新端点未被记录 → 后续开发不知道已有此功能 → 重复造轮子
- Story 状态仍标记 pending → Sprint 规划错误

解决方案：doc-alignment skill（~/.hermes/skills/dogfood/doc-alignment/）
每次完成任务后运行此协议，防止上下文污染累积。
