# SRA-003-15: 质量增强 — 配置验证 + 日志统一 + 魔法数字

## 用户故事

> As a **SRA 系统管理员与维护者**,
> I want **配置有 Schema 校验、日志系统统一输出、匹配分值有命名常量**,
> So that **减少配置错误、提高可调试性、降低维护成本**.

---

## 验收标准

### 📋 AC-1: 配置 Schema 定义
- [ ] 条件: 新增 `~/.sra/config.schema.json` 定义 SRA 配置的完整 JSON Schema
- [ ] 验证: `cat ~/.sra/config.schema.json | python3 -m json.tool` 输出合法 JSON
- [ ] 预期: Schema 覆盖 daemon 现有所有配置项（`http_port`, `log_level`, `skills_dir`, `data_dir`, `runtime_force.level` 等）

### 📋 AC-2: 启动时自动校验配置
- [ ] 条件: SRA Daemon 启动时自动读取 `config.schema.json` 校验 `~/.sra/config.json`
- [ ] 验证: 手动注入非法字段 → 启动日志打印警告但不阻断启动
- [ ] 预期: 非法字段输出 `WARNING` 级别日志说明具体违规项，Daemon 正常启动

### 📋 AC-4: 环境变量覆盖

## 技术要求

- 配置 Schema 使用标准 JSON Schema (draft-07)，不引入额外依赖
- 日志系统使用 Python 标准库 `logging`，不引入第三方日志库
- 环境变量覆盖遵循「小写配置项 → 大写环境变量」映射规则（`http_port` → `SRA_HTTP_PORT`）
- Magic Number 提取为 `IntEnum` 或命名元组，保持向后兼容
- 所有新增配置项都有默认值，不破坏现有用户配置
- `sra config validate` 在 daemon 模式和本地模式均可运行
