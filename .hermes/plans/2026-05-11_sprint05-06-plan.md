## SRA-003-06: 运行时力度体系

### 目标
通过注入覆盖度控制 SRA 介入深度——力度越高，注入点越多。

### 力度层级设计
```
力度不是「阻断强度」，而是「注入覆盖度」：

     用户消息    工具调用前    工具调用后    周期性注入
       ↓           ↓           ↓           ↓
L1 🐣  ●
L2 🦅  ●           ●
L3 🦖  ●           ●           ●
L4 🐉  ●           ●           ●           ●
```

- L1 (basic): on_user_message — 当前行为
- L2 (medium): on_user_message + pre_tool_call (write_file/patch/terminal/execute_code)
- L3 (advanced): on_user_message + pre_tool_call (全部) + post_tool_call
- L4 (omni): 全部 + 周期性重注入每 5 轮

### 实现文件
| 文件 | 变更类型 | 说明 |
|:-----|:--------:|:------|
| `skill_advisor/runtime/force.py` | **新增** | 力度引擎 + 注入点路由 |
| `skill_advisor/runtime/daemon.py` | 修改 | 按力度级别条件启用端点 |
| `skill_advisor/cli.py` | 修改 | `sra config set runtime_force.level` 命令 |
| `skill_advisor/advisor.py` | 修改 | `recheck()` 加固 |
| `tests/test_force.py` | **新增** | 力度体系测试 |

### 验收标准
- [x] 4 级注入覆盖度：basic / medium / advanced / omni
- [x] 每级严格定义对应的注入点集合
- [x] `~/.sra/config.json` 中 `runtime_force.level` 配置
- [x] 默认级别为 `medium`
- [x] `sra config set runtime_force.level advanced` CLI 命令
- [x] 所有注入点均为非阻塞（info/warning 级别，无 block）
- [x] L4 周期性注入间隔可配置（默认 5 轮）
- [x] 测试验证各层级注入点启停
